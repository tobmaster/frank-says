04 / 06

Escalation & Safety

Slow Stop — Coordinator Rules
  • Confidence < 0.7 → Human
  • C-Level involved → Human
  • HR / Finance / Legal → Human
  • Security category → always Human
  • Privileged accounts → Human

Mandate document: 1 page, approved by Legal — what the agent decides alone, what it escalates, what it never touches.

Hard Stop — PreToolUse Hook

Deterministic · No LLM calls · No bypass possible

  • Granting admin rights
  • Password reset for privileged accounts
  • Bulk access changes (> 5 accounts)
  • Any action on a frozen account
Slow Stop Agent pauses, human approves
Hard Stop Tool call blocked — no bypass
Presenter notes
Two-layer safety: the Coordinator makes routing decisions and can escalate (Slow Stop). The hook sits below it — it blocks tool calls before they execute, regardless of what the agent decided (Hard Stop). No LLM in the hook = deterministic behavior, verifiable in code review.

Adversarial Evals

Attack Example

BLOCKED Hook blocks before execution
More Attack Types
  • Prompt injection in the request body
  • False urgency
  • Routine request with legal exposure
  • Ambiguous — multiple categories possible
Scorecard — CI-ready
  • Overall accuracy
  • Precision per category
  • Escalation rate (correct vs. needless)
  • Adversarial pass rate
  • False confidence rate
Test Set Design

Stratified sampling: 20% per category. Runs in CI — the score moves with every change. Legal has a defensible artifact before launch.

Presenter notes
Evals run in CI — not just once. Every change to the Coordinator shifts the score. Stratified sampling prevents easy categories from dominating the overall score. This is the artifact Legal needs to see before approving a go-live.
